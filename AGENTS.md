# Agentic Coding Guidelines - PT Website

This repository contains a **Next.js 16 (App Router)** project using **React 19** and **Tailwind CSS v4**.
The design system is a "Brutalist Editorial" style with strict visual rules.

## 1. Environment & Commands

- **Package Manager**: `npm`
- **Node Version**: v20+
- **Commands**:
  - `npm run dev`    : Start development server
  - `npm run build`  : Production build
  - `npm run start`  : Start production server
  - `npm run lint`   : Run ESLint
  - *Testing*: No test runner is currently configured.
    - If adding tests, prefer **Vitest** for Next.js compatibility.
    - Future command: `npx vitest run path/to/file`

## 2. Code Style & Conventions

### TypeScript & React
- **Strict Mode**: `strict: true` is enabled in `tsconfig.json`. No `any`.
- **Components**: Use Functional Components with named exports.
  ```tsx
  export function MyComponent({ prop }: MyComponentProps) { ... }
  ```
- **Props Interface**: Define interfaces for props, exported if reusable.
- **Hooks**: Use built-in React 19 hooks. Avoid class components.
- **Imports**: Use absolute imports with `@/` alias (e.g., `import { Header } from "@/components/Header"`).

### Naming
- **Components/Files**: PascalCase (e.g., `highlight.tsx` -> `Highlight.tsx` is preferred for components, but current structure uses `src/components/Highlight.tsx`).
- **Functions/Variables**: camelCase.
- **Constants**: UPPER_SNAKE_CASE.

### Error Handling
- Use Error Boundaries for UI recovery.
- Handle async errors with `try/catch` in Server Actions or API routes.
- Use `console.error` for logging in dev, but prefer structured logging in prod.

## 3. Design System (Strict Adherence)

Refer to `DESIGN_SYSTEM.md` for full details.

### Core Principles
- **Square Edges**: **NEVER** use `rounded-*` classes. All buttons, cards, and inputs must have square corners (`border-radius: 0`).
- **Typography**:
  - Headings: **Oswald** (Bold, condensed).
  - Body: **DM Sans** (Clean, geometric).
- **Colors** (Tailwind v4 Variables):
  - Background: `bg-color-background` (`#F4F1EB` - Warm Beige)
  - Foreground: `text-color-foreground` (`#4E4B45`)
  - Accent: `border-color-accent` or `text-color-accent`

### Tailwind v4 Usage
- Use CSS variables defined in `@theme` (see `src/app/globals.css` if available, or implied by `DESIGN_SYSTEM.md`).
- Do not hardcode hex values if utility classes exist.

## 5. Agent Behavior Rules

1. **Check Design First**: Before creating UI, read `DESIGN_SYSTEM.md`. If a requested feature violates "Square Edges" or "Warm Beige" tone, warn the user.
2. **Strict TypeScript**: Do not suppress errors with `@ts-ignore`. Fix the types.
3. **No Fluff**: Write code, not comments describing what the code does (unless complex).
4. **Next.js 16 Specifics**:
   - Use `await params` in dynamic routes (Next.js 15/16 breaking change).
   - Use Server Components by default. Add `"use client"` only when necessary (state, effects).

## 6. Testing Strategy (Future)
